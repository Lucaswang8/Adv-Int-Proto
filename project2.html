Project 2 : Spiral Pattern Generator

An interactive project where you use processing to generate spiral patterns.

Week 1 : Coding the Generator with one external control

The initial code without external controls

Inspiration

I saw a tutorial regarding generative design and they were making a spiral generator so I tried out the coding exercise. However, they were using mouseX as the way to interact with the spiral's form. It did not feel that intuitive for me hence I started to wonder if I could use the potentiometer and other controls to make a spiral pattern generator that I can reuse as a background for my visual designs.

Original Code

Version 1 : Replacing MouseX

I was deciding whether I should replace the mouseX or the "2500" variable. I initially started by replacing mouseX with the serialnumber coming from the Potentiometer

Version 2 : Replacing Variable

I found out that replacing mouseX still made the spiral move but it also reponded to the rotation interaction from the potentiometer. However, I wanted to control the motion entirely hence I multiplied the potentiometer output with the wave variable.

Week 2 : Adding more controls

Changing Colors

I added 4 more controls so that I can adjust the color of the yellow stroke using HSB color mode. I also added a button to trigger noLoop() so that it will stop the spiral. I wanted to be able to screen capture the image. However, I wanted the button to resume the loop but it did not work.

Prototype Demo

Week 3 : Adding more controls and packaging

Controlling the other stroke

I added one more button. Once the button is pressed, it will allow the HSB potentiometers to be used to control the other stroke that was initially white. This way, you can customise the spiral to any colors you choose. I was initially confused on how to set this up. But using if statement and more variables, I was able to make it possible.

Saving Frame

I also changed my noLoop() function to saveFrame. This lets me save my processing image. However, there seems to be an issue capturing. It saves the file but it is blank.

Initial Sketch

I drew this before thinking of adding more controls but I knew from playing with the controls on the bread board that the controls have to be spaced out on the left and right. This is so both hands can be used simultaneously to control it at ease. The controls can't block another hence only by aligning vertically can it work.

Final Package

Initially, I thought that I will have a smaller box but my bread board did not fit. So I had to use a bigger box. Luckily, I had one lying around that was perfect. Thought I had to take out all the controllers and figure out how to rewire them with female to male pins. Some even needed extensions like the button pins.

Reading 2 : What do Prototypes Prototype?

Design systems offer a different way of thinking, where the designer is forced to consider many scenarios and constraints instead of relying on a one-off design process.

Originally I thought design system was mainly a way of saving time to make multiple screens. I thought it only apply to UX Design but reading this made me realise it applies to graphic design and other design fields too. It also made me realise design systems are made to get designers to consider the future rather than just focusing on the present. Not just for the sake of efficiency though it helps too.

The design process is one of trial and error, and it often takes many iterations to create a design that clearly communicates what you're trying to say.

Before, I had the assumption that good designers have natural aesthetic sense but after a few years of designing and seeing my design improve I learnt that design is a process of trial and error. The more you try, the better you get as you build a library of what works and what don't.

Anyone willing to take the necessary trouble will find that, with the aid of the grid system, he is better fitted to find a solution to his design problems which is functional, logical and also more aesthetically pleasing.

I didn't learn about grid system till recently. I used to think it was limiting but the more I use it the more I realise how it does improve our aesthetics overall. Learning coding also made me realise why grid systems are necessary and the art of good design is to balance functionality, logic and aesthetics rather than just one if we want to create sustainable designs (which should be our top priority as interaction designers.)
